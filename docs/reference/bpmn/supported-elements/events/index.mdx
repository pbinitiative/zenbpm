# Events

An event represents something that happens during the course of a process. Events affect the flow of the process and usually have a cause (trigger) or an impact (result).

Green icons are supported and link to their documentation.

## Start events

Start events denote the beginning of a process or subprocess. A Start Event defines the beginning of a process instance: it has no incoming sequence flows and exactly one outgoing sequence flow. It creates a new process instance when triggered, and a process can have multiple start events, each with a different trigger type.

## Intermediate events

Intermediate events occur between the start and end of a process. They are either **catching** — waiting to react to something that happens — or **throwing** — indicating that something has happened and continuing immediately. Catch events pause the flow until the awaited trigger occurs; throw events execute once the token reaches them and continue onward without waiting.

## Boundary events

Boundary events model what should happen if an event occurs while an activity is active. A Boundary Event is attached to the border of an activity and triggers when a specific condition is met while the activity is active, creating an alternative path out of that activity. It can be **interrupting** — the attached activity is cancelled and the token moves along the boundary event's outgoing flow — or **non-interrupting** — the activity continues and a new parallel token is created on the outgoing flow.

## End events

End events denote the end of a particular sequence flow. An End Event marks the completion of a process path and has no outgoing sequence flows. A process may have multiple end events, and the process instance completes once all active tokens have reached an end event.
